# 画像办公室 · 多 Agent 3D 模拟

一个纯前端的 3D 小办公室模拟：多个由「人物画像」驱动的 3D 小人会自主工作、开会、协作讨论、去咖啡角休息。手机和电脑浏览器都能直接打开，无需任何服务器。

## 功能

### 🧠 生成式 Agent（参照斯坦福 Generative Agents）

- **完全隔离的 Agent**：每个人物有自己独立的记忆流（Memory Stream），存浏览器本地、跨天累积。AI 发言是逐句逐人生成的——轮到谁说话，就用"他自己的画像 + 他检索出的记忆 + 他刚听到的话"单独请求模型，没有上帝视角
- **听力范围**：别人说的话只有在物理距离内才会进入一个人的记忆——会议室里大家都听得到，工位闲聊只有旁边的人听到
- **记忆检索**：按 重要度 + 时近性 + 相关性 打分，取最相关的几条进入发言上下文
- **每日反思**：每天下班前每人用模型生成一条当日感悟（🪞），作为高权重记忆影响第二天的言行
- **世界模型**：公司画像（名称/产品/行业/阶段/目标）可在管理后台配置；产品有真实运行状态（日活、满意度、Bug 数、服务器状态、现金跑道），每天自然演化、被市场事件冲击、也被员工行为反向影响（协作讨论有概率修掉 Bug）
- **市场环境**：每天随机发生 1~2 件市场事件（竞品动作、大 V 推荐、服务器故障、安全漏洞、资本市场变化…），以公司公告进入所有人记忆，人物开会时会真的讨论它们
- 左上角**公司仪表盘**实时展示产品指标和今日事件；点击人物可以查看**他自己的记忆流**

### 🏢 基础模拟

- 🏢 **3D 小办公室**：工位区、玻璃会议室（带白板）、咖啡角、休息区，按人数自动布局（最多 8 人）
- 🧑‍💼 **画像驱动的 Agent**：每个人物有姓名、职位、性格画像，点击人物可查看画像卡片，镜头会轻轻跟随
- ⚙️ **管理后台**（`admin.html`，点办公室右上角 ⚙ 进入）：添加/删除人物、编辑画像、调整配色，配置 AI 模型，导入/导出配置
- ✨ **AI 实时对话**（可选）：配置模型后，开会发言和协作讨论由模型根据画像实时生成（日志带 ✨ 标记）；支持 Claude API 和 OpenAI 兼容接口（DeepSeek / Kimi / 通义 / 智谱等）；未配置或调用失败时自动回退内置台词
- 🔒 **隐私**：API Key 与所有配置只存在你浏览器的 localStorage，纯前端直连模型官方接口，没有任何中间服务器
- 📅 **每日日程**：09:00 开工 → 10:00 每日站会（轮流发言）→ 12:00 午休 → 15:30 项目评审会 → 17:30 茶歇 → 18:30 下班，循环往复
- 🤝 **随机协作**：工作时间里人物会随机走到同事工位讨论，对话显示在头顶气泡里
- 🗺️ **A\* 寻路**：人物自动绕开桌椅家具行走，不会穿模卡死
- 📜 **办公室动态**：右下角实时滚动的事件日志
- ⏩ **时间控制**：暂停 / 1× / 3× / 8× 加速
- 📱 **手机适配**：触屏拖动旋转、双指缩放，UI 自动适配小屏

## 如何打开

这是纯静态页面（Three.js 走 CDN），任选一种方式：

**方式一：GitHub Pages（推荐，手机随时可访问）**

仓库 Settings → Pages → 选择分支和根目录，保存后访问分配的网址即可。

**方式二：本地预览**

```bash
npx http-server -p 8080
# 然后浏览器打开 http://localhost:8080
```

> 注意：直接双击 index.html（file:// 协议）无法加载 ES 模块，需要用本地服务器。

**方式三：本地 sidecar（解锁真实市场监控与上层决策）**

```bash
cd sidecar
cp config.example.json config.json   # 改公司简介和 RSS 源
cp .env.example .env                 # 填一个便宜模型的 API Key（筛选新闻用）
npm install && npm start
# 浏览器打开 http://127.0.0.1:7878
```

sidecar 在线时：RSS 新闻经 AI 筛选后实时进入办公室（📡 标记），管理后台可发布「上层决策」长期约束所有人的言行。sidecar 不在线时一切回退纯虚构模式，GitHub Pages 部署不受影响。

> 想让团队讨论你的真实代码：在 `sidecar/config.json` 里把 `repoPath` 设为你的项目目录（绝对路径，或相对 sidecar 的路径，**只读**挂载）。sidecar 会暴露代码检索（tree / file / grep / git log）、每日仓库摘要和静态分析（文件数 / TODO 数 / 复杂度热点），供后续版本的会议讨论引用。安全上：所有路径经白名单校验，拒绝越界与符号链接逃逸，永不执行仓库内脚本。

## 替换成你自己的人物画像

**推荐方式：管理后台。** 打开网页右上角的 ⚙ 进入管理后台，直接添加人物、写画像、选颜色，保存即生效（存在浏览器本地）。同一设备下次打开自动加载；换设备可用「导出/导入配置」迁移。

**配置 AI 模型：** 管理后台 →「AI 模型设置」→ 选提供商、填 API Key 和模型名 → 测试连接 → 勾选启用。
- Claude（Anthropic）：模型名如 `claude-opus-4-8`（省钱可用 `claude-haiku-4-5`），Key 在 [platform.claude.com](https://platform.claude.com) 获取
- OpenAI 兼容接口：填服务商的 Base URL（如 `https://api.deepseek.com/v1`）和模型名（如 `deepseek-chat`）

**代码方式（改默认值）：** 编辑 **`js/personas.js`**。每个人物的格式：

```js
{
  id: "lin",              // 唯一标识
  name: "林晓",            // 姓名
  role: "产品经理",        // 职位
  color: 0xe05a4e,        // 衣服颜色
  skin: 0xf2c9a4,         // 肤色
  hair: 0x2b1d12,         // 发色
  personality: "……",      // 性格描述（点击人物时展示）
  lines: {
    work:    ["…"],       // 工位上的自言自语
    meeting: ["…"],       // 开会发言
    collab:  ["…"],       // 协作讨论台词
    coffee:  ["…"]        // 咖啡角闲聊
  }
}
```

人数可增减（建议 ≤ 6 人，与工位、会议室座位数一致；超出会复用座位）。每日日程也在同一文件的 `DAILY_SCHEDULE` 中调整。

## 目录结构

```
index.html          办公室页面入口
admin.html          ★ 管理后台（人物 + 公司设定 + 模型配置）
css/style.css       办公室 UI 样式（含手机适配）
css/admin.css       管理后台样式
js/personas.js      默认人物画像与日程配置
js/store.js         配置存取（localStorage）
js/admin.js         管理后台逻辑
js/world.js         世界模型：公司画像、产品指标演化、市场事件
js/memory.js        记忆流：每 Agent 独立、重要度/时近性/相关性检索、持久化
js/llm.js           AI 模型客户端（逐人发言 + 每日反思，限流 + 回退）
js/main.js          渲染器、相机、交互、仪表盘、主循环
js/office.js        办公室场景搭建与点位定义（按人数动态布局）
js/agent.js         3D 小人：建模、行走、坐立、气泡、动画
js/director.js      导演：时钟日程 + 隔离式发言 + 听力范围 + 反思 + 世界联动
js/pathfinding.js   网格 A* 寻路
```
