# 画像办公室 · 现实连接版 设计文档

日期：2026-06-12
状态：已与用户逐节确认

## 0. 目标与成功标准

把多 Agent 办公室模拟从纯虚构世界升级为连接现实的闭环系统：

1. 接入实际市场监控（RSS + 搜索引擎实时搜索 + 竞品公开页面 diff 监控）
2. 沙箱挂载只读本地代码仓库，Agent 讨论真实代码问题与市场应对
3. 用户随时注入上层决策（持久政策 + 用户角色化身两种机制）
4. 模拟市场对现有产品和产品改进的反应（闭环）

**成功标准（用户选择"两者并重"）**：演出照旧好看；同时讨论沉淀为可翻阅的产出物（会议纪要、风险清单、市场应对建议、代码问题清单），作为用户的决策参考。

**架构路线（用户选择）**：方案 A 起步（sidecar 当数据面、认知留浏览器）、预留方案 B 接口（认知可迁移至 sidecar）；范围包含完整斯坦福化（Planning / Reacting / 反思树），分期实现。

**时间对齐（用户选择）**：实时推送 + 快进模拟——模拟照常快进，真实事件一到达立刻以"突发新闻"插入当前模拟日，Agent 当场反应；每个新模拟日开工时拉最新快照，无新事件用虚构事件池补位。

## 1. 总体架构

```
┌────────────── 浏览器（认知 + 渲染，现有代码为主）───────────────┐
│ 3D办公室(不动)  Director(改造)  js/cognition/(新增)  admin(扩展) │
└──────┬───────────────┬──────────────────┬──────────────┬──────┘
   SSE 实时推送      快照拉取        仓库工具/embedding    政策CRUD
┌──────┴───────────────┴──────────────────┴──────────────┴──────┐
│              sidecar（Node，新增 sidecar/ 目录，数据面）         │
│ 采集器: RSS │ web search │ 竞品页diff  →  事件总线(SQLite+SSE)  │
│ 仓库服务: 只读 read/grep/log/tree（路径白名单，永不执行仓库代码）│
│ 分析器: 定时 lint/复杂度/TODO → 真实指标 JSON                   │
│ embedding 服务（本地小模型） │ 存储: 政策/产出物/事件历史        │
│ 静态托管前端（替代 npx http-server）                            │
└────────────────────────────────────────────────────────────────┘
```

### 关键设计决定

1. **认知留浏览器，数据面在 sidecar**。Agent 的发言/计划/反思仍由 `js/llm.js` 浏览器直连模型；sidecar 不参与角色对话，只供给真实素材。例外：采集器的相关性筛选与摘要需要 LLM，sidecar 自持一份 API key（`sidecar/.env`，与浏览器 localStorage 那份互不依赖）。
2. **B 接口 = 三份 JSON 契约**：事件格式、记忆格式、产出物格式定义为运行环境无关的 schema；认知模块写成纯函数风格（输入输出均为 JS 对象，不触碰 DOM/localStorage）。
3. **降级永远可用**：sidecar 离线 → 回退现有纯虚构模式；真实事件断供 → 虚构事件池补位；LLM 失败 → 台词池/规则漂移（沿用现有回退模式）。
4. **安全边界**：仓库服务仅暴露 4 个只读端点；`realpath` 前缀校验防 `../` 与符号链接逃逸；服务默认仅绑 `127.0.0.1`；静态分析只用外部只读工具，永不 `npm install`、永不执行仓库自身脚本。

### 市场反应模拟器（闭环核心，浏览器侧，`world.js` 升级）

- **触发**：每模拟日结束跑一次，替换 `nextDay()` 的纯随机漂移。
- **输入**：团队当日行为（修的 Bug、会议决议、行动项完成情况）、产品当前状态（指标 + 近日已上线改进）、当日真实市场事件摘要、现行政策。
- **引擎**：确定性规则做基线演化（便宜稳定）+ 每模拟日 1 次 LLM 调用扮演"市场"，输出结构化 JSON：各指标变化量与原因、2~4 条模拟市场反馈（应用商店评论片段 / 社交帖 / 客户询单——次日作为公告进入全员记忆）、可选竞品跟进动作。
- **落地延迟**：会议拍板的改进经 1~3 模拟日"开发中"才上线，市场再过 1~2 日才显现反应。
- **降级**：LLM 失败回退纯规则漂移。

## 2. sidecar 端点与数据契约

目录：仓库内新增 `sidecar/`（Node ≥ 20，零框架或仅 Express，最少依赖）。

### HTTP API（localhost:7878）

```
GET  /api/snapshot          开工快照：未消费事件 + 最新分析指标 + 现行政策
GET  /api/stream            SSE 实时推送
POST /api/events/ack        标记事件已投递（防重复消费）
GET  /api/repo/tree         目录树（深度限制）
GET  /api/repo/file?path=   读文件（白名单内，截断 N KB）
GET  /api/repo/grep?q=      全文检索（ripgrep，限条数）
GET  /api/repo/log?n=       git log 最近 n 条（含 diff 摘要）
GET  /api/repo/digest       每日仓库动态摘要（缓存，一天一次）
GET  /api/analysis          最新静态分析（lint 错误数、TODO 数、复杂度热点）
POST /api/embed             文本 → 向量（本地模型，批量）
GET/POST/DELETE /api/policies      政策 CRUD
POST /api/artifacts                写产出物
GET  /api/artifacts?type=&day=     翻阅产出物
GET  /                       静态托管前端
```

### 事件契约（三契约之一）

```json
{
  "id": "evt_a1b2c3",
  "ts": 1781234567890,
  "source": "rss | search | watch | manual | policy",
  "kind": "market",
  "title": "夸克网盘上线新一轮校园活动",
  "summary": "≤120字，LLM 生成，写入记忆的内容",
  "url": "原文链接",
  "relevance": 8,
  "suggestedImpact": { "sat": -1, "dau": "+1%" },
  "consumed": false
}
```

`relevance`（0~10，LLM 按公司画像打分）低于阈值直接丢弃；进模拟的每天应只有 3~5 条。`suggestedImpact` 仅供市场反应模拟器参考，不直接改指标。

### 采集器

| 采集器 | 节奏 | 流程 |
|---|---|---|
| RSS | 每 30 分钟 | 拉源 → URL hash 去重 → LLM 批量打分 + 摘要 → 入库 |
| web search | 每天 2 次 | 关键词组（admin 可配）→ Claude web search → 同上 |
| 竞品页 diff | 每 2 小时 | 抓取配置 URL → 正文提取 → 与上次快照比对 → 有实质变化才 LLM 总结成事件 |

### 存储

单文件 SQLite：事件、政策、产出物、页面快照四张表。选 SQLite 而非 JSON 文件：事件需按 `consumed` 查询与去重；不选 Postgres：单用户本地工具。

## 3. 前端认知层（斯坦福化）

新增 `js/cognition/`：`plan.js`、`react.js`、`reflect.js`、`converse.js`，全部纯函数风格（B 接口模块）。Director 从「编剧」降级为「物理引擎 + 日程骨架 + 路由」。

### plan.js — 每日计划

每天开工每人 1 次调用。输入：画像 + 昨日反思 + 今晨快照（真实事件、政策、仓库摘要、昨夜市场反馈）+ 名下未完成行动项。输出：

```json
{ "intentions": [
  { "slot": "上午", "what": "查 grep 里那个限速逻辑的 TODO", "with": null, "kind": "investigate" },
  { "slot": "下午", "what": "找老王对带宽成本方案", "with": "王强", "kind": "collab" }
]}
```

计划存为高权重记忆并挂在 agent 上；协作配对从 intentions 优先取（`director.js` 的随机配对降为兜底），发言场景携带当前意图。

### react.js — 突发反应

SSE 事件白天到达 → embedding 计算事件与每人画像相似度（本地免费）→ top 2 各 1 次小调用，输出 `{utterance, action}`，action ∈ `goto_colleague / call_meeting / investigate_repo / none`；其余人仅写入公告记忆。保持逐人独立判断的隔离原则，控制调用量。

### reflect.js — 反思树

替换 17:50 定时一句话。**重要度累积触发**：自上次反思以来新记忆重要度之和超过阈值（约 50）→ 模型从近期记忆提出 2~3 个问题 → 逐题检索、生成带证据引用的洞见（`evidence: [记忆id]`）→ 反思可被后续反思引用，形成层级。下班兜底跑一次。

### converse.js — 真多轮对话

协作讨论从写死的三轮改为循环：每轮发言者输出 `{utterance, done}`，自行决定结束，上限 6 轮。会议保留轮转但允许跳过（"没什么要补充"）。**会议结束 1 次调用生成结构化纪要**（决议/风险/行动项）→ POST 产出物库 → 行动项写入负责人记忆，成为次日计划输入。

### 行动项生命周期

`待办 → 开发中(1~3 模拟日) → 已上线`；上线改进进入市场反应模拟器输入，市场反馈次日回流。

### memory.js 升级

- 相关性：bigram → embedding 余弦（向量经 sidecar `/api/embed`）
- 时近性：`Date.now()` → 模拟时间（`day * 570 + clockMin`），修复加速/隔夜衰减错乱
- 持久化：sidecar 在线存 SQLite（容量大、可存向量），离线回退 localStorage（无向量，退回 bigram）。记忆 JSON 为三契约之二。

### llm.js 扩展

`speak()` 可选携带仓库工具（repo_grep / repo_read / repo_log）做 tool-use 循环（上限 2 轮），技术讨论时发言可引用真实文件与代码行。

### 成本账（8 人，每模拟日）

计划 8 + 反应 4~6 + 会议发言（同现状）+ 反思 8~16 + 纪要 2 + 市场裁判 1 ≈ 现状 2~3 倍，由现有 `usage` 档位统一节流。

## 4. 决策注入、用户化身、admin、分期

### 政策注入（持久政策 + 公告）

政策 `{id, text, issuedDay, active}` 存 sidecar 政策表。发布：全员收到重要度 9 公告记忆一次；此后所有 speak/plan/reflect 的 system prompt 注入「现行公司政策」区块直至撤销；撤销亦广播公告。

### 用户角色化身

用户以可配置身份（默认"董事长"）拥有 3D 化身（复用 `Agent` 类，去掉自主行为）：

- **单独面谈**：点击 Agent → 化身走过去 → DOM 对话框。用户输入进入对方记忆（听力范围内他人同样听到，复用 `broadcastHearing`）；回复走现有 `speak()`，场景为"董事长来找你面谈"。即论文的「采访 Agent」。
- **参加会议**：会议中用户可发言，以会议听力半径广播，重要度 8。

### admin 扩展

四个新区块：上层决策（政策 CRUD）、数据源（RSS 源 / 搜索关键词组 / 竞品 URL / 仓库路径）、sidecar 状态（连接、采集器最近运行、今日事件数）、产出物（按日翻阅纪要/日报/市场反馈）。仪表盘：真实事件标 📡、虚构补位标 🎭，指标注明来源。

### 分期（每期独立可用）

| 期 | 内容 | 交付价值 |
|---|---|---|
| P1 | sidecar 骨架 + SQLite + RSS 采集器 + 事件契约 + SSE/快照 + 政策注入 + admin 政策页 | 真实新闻进办公室，决策长期生效 |
| P2 | 仓库 4 端点 + 每日摘要 + 静态分析 + search/diff 采集器 + embedding 服务 + memory 升级 + 会议 tool-use | Agent 开会引用真实代码，检索质变 |
| P3 | plan/react/reflect/converse + 会议纪要 + 行动项状态机 + 市场反应模拟器 | 完整闭环：讨论→改进→市场反应→再讨论 |
| P4 | 用户化身 + 面谈/参会 + 产出物浏览 UI + 仪表盘扩展 | 用户进入这个世界 |

## 5. 测试策略

沿用现有 `test-*.mjs` 模式：

- sidecar 单测：路径逃逸防护、事件去重、契约校验
- 认知模块（纯函数）：mock 模型输出，测计划解析、反思触发阈值、对话终止条件
- 集成：用 huaxiang 仓库自身作为被挂载的"产品代码"
- 降级冒烟：sidecar 离线 / LLM 离线 各一条

## 6. 错误处理总则

每层都有降级路径且模拟永不停止：采集器失败静默重试并在 admin 状态面板暴露；LLM 失败沿用现有冷却 + 回退机制；sidecar 不可达时前端定期探测自动重连，期间走虚构模式。
